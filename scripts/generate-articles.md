# 記事生成ワークフロー

## 概要

選定されたニュース候補から記事を生成し、セルフレビューを行う。

## 実行タイミング

`gather-news.md` の Step 5 完了後に実行

## 入力

`gather-news.md` で選定された 5〜10 件のニュース候補リスト

## 手順

### Step 1: 記事生成

各候補について以下を実行:

1. ソース記事の内容を Web 検索で深掘りする（追加情報・関連事例を収集）
2. バーティカル YAML の `editorial` セクションを参照し、記事の角度を決定する
3. 記事テンプレートに従って記事を生成する

### 記事テンプレート

```markdown
---
title: "タイトル（20〜35字、問いかけ型 or キーワード提示型）"
vertical: "{バーティカルslug}"
source_url: "{ソースURL}"
source_name: "{メディア名}"
published_date: YYYY-MM-DD
status: "review"
quality_tier: "{brief|standard|deep}"
tags: ["タグ1", "タグ2"]
summary: "1〜2文の要約（100字以内）"
---

## ニュースの概要と背景

（300〜500字: 何が起きたか、誰が関わっているか、前提となる文脈）

## なぜ注目すべきか

（200〜400字: ベタービジネス/エコシステムの文脈からの意味づけ）

## 関連する動向・事例

（200〜400字: 国内外の関連事例、類似の取り組み）

## 読者への示唆

（150〜250字: 実践者として何を考え、どう活かせるか。問いかけで締める）
```

### 文体ルール

| 項目 | 規定 |
|------|------|
| 文末 | だである調（常体） |
| 人称 | 三人称基本 |
| 語り口 | 落ち着いた、知的好奇心のある |
| テンション | 中庸 |
| 総文字数 | brief: 500-800字 / standard: 1000-1500字 / deep: 1500-2500字 |

### 表記ルール

- AI表記: 「AI」に統一
- 英数字: 半角
- カッコ: 「」会話・引用、『』書名
- 三点リーダー: 「……」二つ連続
- ダッシュ: 「——」二つ連続
- 読点: 「、」/ 句点: 「。」

### メタファールール

**使ってよい**: ガーデニング/生態系メタファー
- 種をまく、芽が出る、根を張る、土壌を耕す、生態系を育む、共生する

**禁止**: 戦争系・機械系・競争系メタファー

### Step 2: セルフレビュー

各記事に対して以下のチェックリストで自動検証:

- [ ] 固有名詞（人名・社名・地名）は正確か
- [ ] 数値・日付は確認済みか
- [ ] バーティカルの `editorial.angle` に沿っているか
- [ ] `editorial.forbidden_framing` に違反していないか
- [ ] だである調で統一されているか
- [ ] 禁止メタファーを使用していないか
- [ ] 文字数が `article_length` の範囲内か
- [ ] タイトルが20〜35字の範囲か
- [ ] リード文（summary）が記事内容を的確に要約しているか

### Step 3: 品質スコアリング

チェック結果に基づいて品質スコア（0-100）を算出:

| チェック項目 | 配点 |
|-------------|------|
| 事実性 | 20 |
| コンセプト整合性 | 20 |
| ブランドボイス | 15 |
| 表記・文体 | 15 |
| 構成・読みやすさ | 15 |
| 読者価値 | 15 |

### Step 4: ファイル出力

生成した記事を以下のパスに Markdown ファイルとして保存:

```
src/content/articles/YYYY-MM-DD-{slug}.md
```

slug はタイトルから生成（英語のキーワードベース、ハイフン区切り）。

## 出力

- 生成された記事ファイル（5〜10件）
- 各記事の品質スコアと品質ティア分類
- `batch-review.md` ワークフローに渡すサマリーデータ
