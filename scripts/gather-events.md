# イベント収集・記事生成ワークフロー

## 概要

全アクティブイベントカテゴリに対して並行でイベント検索を行い、候補リストを生成し、紹介記事の下書きまで自動生成する。

## 実行タイミング

毎日 7:00（gather-news.md と同タイミング、scheduled task）

## 手順

### Step 1: イベントカテゴリ定義の読み込み

`src/content/event-verticals/` 配下の全 YAML ファイルを読み込み、`active: true` のカテゴリのみを対象とする。

### Step 2: 並行イベント検索

各カテゴリについて以下を実行:

1. YAML の `event_sources` に定義された各プラットフォームの検索クエリで Web 検索を実行する
2. 各カテゴリ 3〜5 件の候補を収集する
3. 過去のイベント（event_date < today）を除外する
4. 既存 `src/content/events/` との重複を除外する（イベント名+日付で判定）

#### プラットフォーム別の検索方法

**Peatix / connpass / EventBrite**:
- WebSearch で `site:peatix.com {query}` / `site:connpass.com {query}` 等のサイト指定検索
- 発見したURLを WebFetch で取得し、イベント詳細を抽出

**PR TIMES**:
- WebSearch で `site:prtimes.jp {query}` でプレスリリース内のイベント告知を検索
- 発見したURLから WebFetch でイベント情報を抽出

**WebSearch（汎用）**:
- クエリに年号を含めて検索し、主催者サイト上のイベントページを発見
- 発見したURLから WebFetch で詳細取得

#### 各イベントから抽出する情報

| 項目 | 必須/任意 | 備考 |
|------|----------|------|
| イベント名 | 必須 | 正式名称 |
| 開催日時 | 必須 | 日付・時間・タイムゾーン |
| 終了日時 | 任意 | わかれば |
| 会場/形式 | 必須 | オンライン/オフライン/ハイブリッド |
| 主催者 | 必須 | 団体名・個人名 |
| 登壇者・ゲスト | 任意 | フルネーム＋肩書き |
| 参加費 | 任意 | 無料/有料（金額） |
| 申込URL | 必須 | 有効なリンク |
| イベント概要 | 必須 | テーマ・内容の説明 |
| ソースURL | 必須 | 元のイベントページURL |
| ソースプラットフォーム | 必須 | Peatix / connpass 等 |

### Step 3: スコアリング

各イベント候補をカテゴリYAMLの `selection_weights` に基づいてスコアリングする。

**スコア計算**:
```
total_score = timeliness × weight_timeliness
            + reader_value × weight_reader_value
            + concept_alignment × weight_concept_alignment
            + uniqueness × weight_uniqueness
```

**各スコアの基準（0〜100）**:

| スコア | 基準 |
|--------|------|
| timeliness | 開催日までの距離: 1〜2週間先=90, 3〜4週間先=70, 1〜2ヶ月先=50, 3ヶ月以上先=30 |
| reader_value | プライマリペルソナ「探究する実践者」（30〜50代、経営者/事業開発/NPO）への価値 |
| concept_alignment | ベタービジネス4次元（自己充足的/経済的/社会的/文化的）との接続度 |
| uniqueness | 他メディアでの紹介状況、類似イベントとの差別化度 |

**閾値**: `total_score >= 60` のイベントのみ候補リストに残す。

### Step 4: 候補リストの生成

収集・スコアリングした候補を以下のフォーマットで整理する:

```json
{
  "date": "YYYY-MM-DD",
  "candidates": [
    {
      "category": "zebra-bcorp",
      "event_name": "イベント名",
      "event_date": "YYYY-MM-DD",
      "event_date_end": "YYYY-MM-DD",
      "venue_name": "会場名",
      "venue_type": "offline",
      "organizer": "主催者名",
      "speakers": ["登壇者1（肩書き）", "登壇者2（肩書き）"],
      "fee": "無料",
      "registration_url": "https://...",
      "source_url": "https://...",
      "source_name": "Peatix",
      "description": "イベント概要",
      "total_score": 85,
      "scoring_rationale": {
        "timeliness": 90,
        "reader_value": 80,
        "concept_alignment": 85,
        "uniqueness": 75
      },
      "better_business_connection": "社会的次元: コミュニティベースの経済モデルを議論"
    }
  ]
}
```

### Step 5: 補足リサーチ

スコア上位のイベント（1日1〜3件目安）について、記事の深みを出すための追加調査を行う:

**ツール**: WebSearch

1. **主催者の背景**: 組織のミッション、過去の活動、関連プロジェクト
2. **登壇者のプロフィール**: 専門領域、著書・発信、これまでの実績
3. **テーマの文脈**: なぜ今このテーマが重要か、関連する社会動向や議論

### Step 6: 記事下書き生成

各候補について、以下のテンプレートに従い記事下書きを自動生成する。

#### 記事テンプレート

```markdown
---
title: "記事タイトル（20〜35字、問いかけ型 or キーワード提示型）"
vertical: "{カテゴリslug}"
source_url: "{ソースURL}"
source_name: "{ソースプラットフォーム名}"
published_date: YYYY-MM-DD
status: "review"
quality_tier: "standard"
tags: ["カテゴリタグ", "テーマタグ", "イベント"]
summary: "要約（100字以内）"
event_name: "イベント正式名称"
event_date: YYYY-MM-DD
event_date_end: YYYY-MM-DD
venue_name: "会場名 or オンライン"
venue_type: "offline"
organizer: "主催者名"
registration_url: "https://..."
fee: "無料"
---

## イベントの概要

（300〜500字）
- イベントの目的・背景
- 主催者の紹介と活動文脈
- 登壇者・プログラムの概要

## なぜ今、このテーマか——ベタービジネスの文脈から

（300〜500字）
- テーマの社会的背景・時代的意義
- ベタービジネス4次元との接続
- inquire.jp 読者にとっての意味

## 注目のポイント

（300〜500字）
- このイベントならではの見どころ
- 登壇者の専門性やユニークな視点
- 期待される議論や学び

## 開催情報

- **イベント名**: 正式名称
- **日時**: YYYY年MM月DD日（曜日）HH:MM〜HH:MM
- **会場**: 会場名（住所） or オンライン（使用ツール）
- **参加費**: 無料 or 金額
- **定員**: XX名（わかれば）
- **申込**: [申込ページ](URL)

### 参考リンク

- [主催者公式サイト](URL)
- [関連記事・情報](URL)
```

**総文字数**: 1500〜2500字

#### 文体ルール

| 項目 | 規定 |
|------|------|
| 文末 | だである調（常体） |
| 人称 | 三人称基本 |
| 語り口 | 落ち着いた、知的好奇心のある |
| テンション | 中庸。過度な興奮も冷淡さも避ける |

#### イベント紹介記事特有の注意点

- **未来形を基調とする**: 開催前のため「開催される」「議論される予定である」等の表現を使う
- **過度な期待を煽らない**: 「必見」「絶対に参加すべき」等の表現は避ける
- **事実と展望を区別する**: 確定情報（日時・会場・登壇者）と、見立て（注目ポイント・期待される議論）を明確に分ける

### Step 7: セルフレビュー

各記事に対して以下のチェックリストで自動検証:

- [ ] 固有名詞（人名・団体名・イベント名）は正確か
- [ ] 日時・会場・参加費は正確か
- [ ] 申込URLは有効か（WebFetchで確認）
- [ ] 開催日が未来の日付であるか
- [ ] カテゴリYAMLの `editorial.angle` に沿っているか
- [ ] `editorial.forbidden_framing` に違反していないか
- [ ] だである調で統一されているか
- [ ] 禁止メタファー（戦争系・機械系・競争系）を使用していないか
- [ ] 未開催のイベント内容を事実として断定していないか
- [ ] 過度な参加促進表現がないか
- [ ] 文字数が1500〜2500字の範囲か
- [ ] タイトルが20〜35字の範囲か

#### 品質スコアリング

チェック結果に基づいて品質スコア（0-100）を算出:

| チェック項目 | 配点 |
|-------------|------|
| 事実性 | 20 |
| コンセプト整合性 | 20 |
| ブランドボイス | 15 |
| 表記・文体 | 15 |
| 構成・読みやすさ | 15 |
| 読者価値 | 15 |

品質スコア70以上のもののみファイル出力する。70未満は再生成を試みる。

### Step 8: ファイル出力

生成した記事を以下のパスに Markdown ファイルとして保存:

```
src/content/events/YYYY-MM-DD-{slug}.md
```

slug はイベント名から生成（英語キーワードベース、ハイフン区切り）。

### Step 9: バッチレビュー依頼

生成された下書きのサマリーをユーザーに提示:

```
## イベント紹介記事 下書き完了（MM/DD）

### 新規 X件

1. **記事タイトル**
   - イベント: イベント名（MM/DD開催）
   - カテゴリ: カテゴリ名
   - 品質スコア: XX/100
   - ステータス: review

2. ...

→ 確認して承認/修正/除外を指示してください。
  承認されたものは status を "approved" に変更します。
```

ユーザーが承認したら、該当記事の frontmatter `status` を `"approved"` に変更する。

## 注意事項

- 著作権に配慮し、イベント説明文の引用は最小限にすること
- 申込URLが有効であることを必ず確認すること
- 同日に同じイベントを複数カテゴリから重複して取り上げないこと
- 開催日が過ぎたイベントは収集対象から除外すること
- ニュース記事（gather-news.md）で同じトピックを取り上げている場合は、重複を避ける方向で調整すること
